# Security Policy

## Supported Versions

We take security seriously. We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Qevora, please send an e-mail to security@qevora.com. All security vulnerabilities will be promptly addressed.

Please do **NOT** report security vulnerabilities via public GitHub issues.

### What to include in your report
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact of the vulnerability.
- Any suggested mitigations (optional).

You should receive a response within 48 hours. If the issue is confirmed, we will release a patch as soon as possible.
